# 技术文档 - 光伏行业财务数据爬取系统

## 1. 技术选型

### 1.1 数据源：AKShare (东方财富接口)

选择 AKShare 作为数据获取库，理由：
- **免费开源**：无需注册、无需API Key
- **数据可靠**：底层调用东方财富官方数据接口，数据源自交易所公告
- **覆盖完整**：支持A股全部上市公司，历史数据可追溯至2007年
- **维护活跃**：持续更新，社区活跃

备选方案对比：

| 方案 | 免费 | 需注册 | 覆盖度 | 适用性 |
|------|------|--------|--------|--------|
| **AKShare** | 是 | 否 | 完整报表 | **最佳** |
| Tushare | 否(需积分) | 是 | 完整报表 | 积分门槛高 |
| efinance | 是 | 否 | 仅摘要 | 缺少总资产/总负债 |
| Baostock | 是 | 否 | 仅比率 | 无绝对值数据 |

## 3. 核心API调用

### 3.1 资产负债表
```python
import akshare as ak
# symbol格式: SH601012 (上交所) / SZ002459 (深交所)
df = ak.stock_balance_sheet_by_report_em(symbol="SH601012")
```
关键字段：
- `REPORT_DATE`: 报告日期 (如 2024-03-31)
- `TOTAL_ASSETS`: 总资产
- `TOTAL_LIABILITIES`: 总负债

### 3.2 利润表
```python
df = ak.stock_profit_sheet_by_report_em(symbol="SH601012")
```
关键字段：
- `REPORT_DATE`: 报告日期
- `OPERATE_INCOME`: 营业收入
- `NETPROFIT`: 净利润
- `PARENT_NETPROFIT`: 归属母公司净利润

## 4. 数据处理流程

### 4.1 爬取阶段
1. 遍历30家公司
2. 对每家公司分别调用资产负债表和利润表API
3. 每次请求间隔1-2秒，避免触发反爬
4. 请求失败时最多重试3次，使用指数退避

### 4.2 清洗阶段
1. 从原始数据中提取目标字段
2. 按 `REPORT_DATE` 过滤季度报告（03-31, 06-30, 09-30, 12-31）
3. 过滤时间范围：2019-01-01 至 2025-09-30
4. 合并资产负债表和利润表数据（按公司+报告日期关联）
5. 生成季度标签（如 2019Q1, 2019Q2...）

### 4.3 导出阶段
- 导出汇总Excel文件，包含所有公司所有季度的数据
- 同时导出CSV格式便于程序读取

## 5. 错误处理策略

- **网络超时**: 重试3次，间隔2/4/8秒
- **数据缺失**: 记录日志，对应单元格填充NaN
- **API异常**: 捕获异常，跳过该公司并记录，继续处理下一家
- **字段变更**: 使用字段映射配置，便于适配API变更

## 6. 性能预估

- 每家公司需调用2个API（资产负债表 + 利润表）
- 每次API调用约3-7秒（含分页）
- 30家公司总计约60次API调用
- 预计总运行时间：4-8分钟
